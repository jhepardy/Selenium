Behaviour notes
Parallel + retry: With TestNG 7.10, parallel="methods" plus IRetryAnalyzer led to a GraphOrchestrator NPE. Default smoke suite is sequential (parallel="none"). Use parallel-regression.xml against Grid for parallel runs; if issues persist, set retry.max.attempts=0 in properties or upgrade TestNG when a fix is available.
Env / CI: SELENIUM_GRID_URL and BROWSER are read from the environment before applying -D overrides (EnvironmentConfig).
Jenkins: GRID_URL parameter maps to SELENIUM_GRID_URL for the Maven step (no credential binding required for a minimal example).
Run
cd /Users/user/dev/projects/Selenium
mvn clean test -Denv=dev -Dheadless=true -Dbrowser=chrome
Smoke suite is the default in pom.xml; mvn clean test already runs it. mvn clean test completed successfully here (3 smoke tests against Heroku’s demo app).

Excel: ExcelDataLoader is ready; add a .xlsx under src/main/resources/testdata/ and a test DataProvider when you need it. JDBC: set jdbc.url / jdbc.user / jdbc.password and use JdbcCredentialsLoader.loadFromQuery(...) against a test database only.
